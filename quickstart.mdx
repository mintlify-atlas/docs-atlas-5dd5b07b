This guide walks you through a complete working setup: logging to the console and a file simultaneously using Puppy's two most common transports. By the end you'll have a `Puppy` instance ready to use, you'll know all eight log levels and when to reach for each, and you'll understand how to adjust the logger pipeline at runtime.

<Note>
  This guide assumes you've already added Puppy to your project. If not, see the [installation guide](/installation) first.
</Note>

## Set up console and file logging

<Steps>
  <Step title="Import Puppy">
    Add the import at the top of your Swift file:

    ```swift
    import Puppy
    ```
  </Step>
  <Step title="Create a console logger">
    `ConsoleLogger` writes to standard output. The first argument is a unique reverse-domain label—Puppy also uses it as the `DispatchQueue` label internally, so it must be unique per logger instance.

    ```swift
    let console = ConsoleLogger("com.example.yourapp.console", logLevel: .info)
    ```

    Any message below `.info` (i.e. `.trace`, `.verbose`, `.debug`) will be silently dropped by this logger.
  </Step>
  <Step title="Create a file logger">
    `FileLogger` appends messages to a file. Provide an absolute URL and an optional Unix permission mode string (defaults to `"640"`).

    ```swift
    let fileURL = URL(fileURLWithPath: "./foo.log").absoluteURL
    let file = FileLogger("com.example.yourapp.file",
                          logLevel: .info,
                          fileURL: fileURL,
                          filePermission: "600")
    ```
  </Step>
  <Step title="Combine loggers into a Puppy instance">
    Pass both loggers to `Puppy`. Every call on the `puppy` instance is forwarded to all registered loggers concurrently.

    ```swift
    var puppy = Puppy(loggers: [console, file])
    ```
  </Step>
  <Step title="Log messages">
    Call any of the eight log-level methods. Each logger applies its own minimum level filter independently:

    ```swift
    puppy.trace("TRACE message")    // dropped — below .info on both loggers
    puppy.verbose("VERBOSE message") // dropped
    puppy.debug("DEBUG message")    // dropped
    puppy.info("INFO message")      // logged to console and file
    puppy.notice("NOTICE message")  // logged to console and file
    puppy.warning("WARNING message") // logged to console and file
    puppy.error("ERROR message")    // logged to console and file
    puppy.critical("CRITICAL message") // logged to console and file
    ```
  </Step>
</Steps>

## Complete example

Here is the full setup from the steps above in a single block:

```swift
import Puppy

let console = ConsoleLogger("com.example.yourapp.console", logLevel: .info)
let fileURL = URL(fileURLWithPath: "./foo.log").absoluteURL
let file = FileLogger("com.example.yourapp.file",
                      logLevel: .info,
                      fileURL: fileURL,
                      filePermission: "600")  // Default permission is "640".

var puppy = Puppy(loggers: [console, file])

puppy.debug("DEBUG message")  // Will NOT be logged.
puppy.info("INFO message")    // Will be logged.
puppy.error("ERROR message")  // Will be logged.
```

## Log levels

Puppy defines eight levels as a `UInt8` enum in ascending severity order. A logger configured at a given level drops all messages with a lower raw value.

| Method | Level | Raw value |
|--------|-------|-----------|
| `puppy.trace(...)` | `.trace` | 1 |
| `puppy.verbose(...)` | `.verbose` | 2 |
| `puppy.debug(...)` | `.debug` | 3 |
| `puppy.info(...)` | `.info` | 4 |
| `puppy.notice(...)` | `.notice` | 5 |
| `puppy.warning(...)` | `.warning` | 6 |
| `puppy.error(...)` | `.error` | 7 |
| `puppy.critical(...)` | `.critical` | 8 |

## Adding and removing loggers at runtime

Because `Puppy` is a `struct`, you need a `var` binding to mutate it. Use `add(_:)` and `remove(_:)` to update the logger pipeline at runtime. Loggers are identified by their label—adding a logger whose label already exists is a no-op.

```swift
var puppy = Puppy()

let console = ConsoleLogger("com.example.yourapp.console", logLevel: .debug)
puppy.add(console)

puppy.debug("Now logging to console")

// Remove the console logger later
puppy.remove(console)

// Remove all loggers at once
puppy.removeAll()
```

<Tip>
  Use reverse-domain labels like `com.example.yourapp.console` to keep them globally unique. Puppy uses the label as the underlying `DispatchQueue` label too.
</Tip>

## Flushing pending log writes

Each logger dispatches messages asynchronously on its own queue. Call `flush()` to wait until all pending messages have been processed—useful before your application exits or suspends.

```swift
let result = puppy.flush()  // waits up to 3 seconds by default

switch result {
case .success:
    print("All messages flushed.")
case .timeout:
    print("Flush timed out.")
}
```

You can pass a custom timeout in seconds:

```swift
let result = puppy.flush(5.0)  // wait up to 5 seconds
```

`flush(_:)` returns a `WaitingResult` with two cases: `.success` and `.timeout`.

<Warning>
  Do not call `flush()` from within a logger's dispatch queue or a queue that a logger depends on—it will deadlock. Call it from your application's main thread or a dedicated shutdown context.
</Warning>
