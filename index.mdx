Puppy is a flexible logging library written in Swift that runs on Darwin (macOS, iOS, tvOS, watchOS, visionOS), Linux, and Windows. It lets you route log output to multiple transports simultaneously—console, file, rotating files, syslog, or Apple's unified logging system—and plug directly into the `apple/swift-log` ecosystem as a backend.

## What Puppy does

Puppy routes log messages to one or more transports at once. Each transport is an independent logger with its own log level, format, and dispatch queue. You compose them into a single `Puppy` instance and call one API to log everywhere.

## Key features

- **Multiple transports** — Log to console, file, rotating file, syslog (Linux), and OSLog (Darwin) at the same time
- **File log rotation** — Rotate by file size; archive with sequential numbering or date-UUID suffixes
- **Cross-platform** — Darwin (macOS, iOS, tvOS, watchOS, visionOS), Linux, and Windows
- **swift-log backend** — Drop-in `PuppyLogHandler` for projects already using `apple/swift-log`
- **Customizable formatting** — Implement `LogFormattable` to produce any log line structure
- **Extensible** — Implement `Loggerable` to create custom transports
- **8 log levels** — `trace`, `verbose`, `debug`, `info`, `notice`, `warning`, `error`, `critical`
- **Thread-safe** — Each logger runs on its own `DispatchQueue`

## Get started

<Steps>
  <Step title="Add Puppy to your package">
    Add the dependency to your `Package.swift`:
    ```swift Package.swift
    .package(url: "https://github.com/sushichop/Puppy.git", from: "0.9.0")
    ```
  </Step>
  <Step title="Import and configure loggers">
    Create one or more loggers and pass them to `Puppy`:
    ```swift
    import Puppy

    let console = ConsoleLogger("com.example.app.console", logLevel: .info)
    let puppy = Puppy(loggers: [console])
    ```
  </Step>
  <Step title="Log messages">
    Call any of the eight log-level methods:
    ```swift
    puppy.info("Application started")
    puppy.warning("Disk space below threshold")
    puppy.error("Failed to connect to server")
    ```
  </Step>
</Steps>

<Tip>
  If your project already uses `apple/swift-log`, see the [swift-log integration guide](/guides/swift-log-integration) to use Puppy as a backend without changing your logging calls.
</Tip>
